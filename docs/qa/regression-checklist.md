# 회귀 체크리스트 (regression-checklist)

> 매 PR 의 단계 [7] (TM 2차 종합) 직전, 해당 PR 의 변경 범위에 따라 본 체크리스트를 점검합니다. 항목이 비어있는 카테고리는 슬라이스가 도착하기 전까지 N/A. 새로운 슬라이스가 도착하면 해당 카테고리를 채워주세요.

## 0. 공통 (모든 PR)

- [ ] CI 모든 필수 워크플로우 통과
- [ ] PR 본문 / 워크플로우 단계 체크리스트 일관
- [ ] 변경 파일이 PR 설명과 일치
- [ ] 시크릿 / `.env` 커밋 없음 (GitGuardian 통과)
- [ ] 문서 동기화 (`docs/` 갱신 필요했다면 함께 반영)
- [ ] 한글 외 PR/리뷰 코멘트 없음

## 사용 방법

- 슬라이스가 머지될 때마다 위 카테고리에 항목을 **추가** 합니다 (제거가 아닌 누적).
- TM 이 단계 [6a]/[6b] / PM 이 단계 [7] 에서 본 파일을 열어 변경 범위에 해당하는 항목을 점검.

### 항목 분류 (N/A vs 블로커)

| 카테고리 | 분류 |
| --- | --- |
| **§0 공통** | **필수** — 미체크 시 [블로커] |
| §1 클라이언트 | 해당 슬라이스 머지 전: N/A. 머지 이후 PR: 변경 영향 있을 때만 필수 |
| §2 서버 | 동일 — PR #7 머지 이전 N/A, 이후 변경 영향 시 필수 |
| §3 인프라 | PR #1 부터 일부 필수 (`docker compose up -d` 정상 부팅), 나머지는 PR #7 이후 |
| §4 데이터 / 콘텐츠 | PR #4 머지 이전 N/A |
| §5 스토리 / 퀘스트 | PR #8 머지 이전 N/A |
