# PR #23 기획서 — 밸런스 시뮬레이터 + 보정 패스 (M6)

> M6 출시 준비 중 **자율 가능** 항목. 밸런스 문서가 전반에 걸쳐 "M2 시뮬레이터 도착 시 보정" 으로 미뤄둔 1차 수치를, 실제 시뮬레이션으로 검증/보정한다. `tools/balance-sim/` 신설. 사운드/Steam SDK 는 외부 의존이라 별도 후속.

## 1. 목표 / DoD
`tools/balance-sim/` 가 기존 공식(level/combat/stats/offline)을 재사용해 플레이어 진행을 시뮬레이션하고, **1000회 시뮬로 환생 도달 시간 분포**(타깃 5~10h)를 산출한다. 결과로 1차 수치(EXP 곡선 지수, 골드/시간, 오프라인 효율 등)의 타깃 충족 여부를 리포트하고, 벗어나면 보정안을 제시·적용한다.

### DoD 검증
1. `tools/balance-sim/` 실행 → 환생(보스격파+Lv100) 도달 시간 분포(중앙값/분위) 출력.
2. 타깃(5~10h) 대비 현 1차 수치 평가 리포트(JSON/md).
3. 벗어난 항목은 보정안 제시 + (합의 가능 범위) 공식 상수 보정 + 단위테스트 갱신.
4. `npm test`(시뮬 단위테스트 포함) + tsc + lint GREEN. 기존 게임 테스트 회귀 없음.

## 2. 범위 (In Scope)
### 2.1 시뮬레이터 (메인, TS)
- `tools/balance-sim/`: 기존 `server/src/core/formulas/` (level.ts EXP/누적, combat.ts DPS, stats.ts 직업 성장, offline.ts) 재사용(import).
- 모델: 시간 진행 → 사냥 DPS 기반 킬/시간 → EXP/골드 누적 → 레벨업 → (장비/펫/스킬 보너스 근사) → Lv100+보스 격파 시점 = 환생 도달 시간. 오프라인 비중 반영.
- 1000회(파라미터 변동/난수) 시뮬 → 도달 시간 분포(median/p10/p90).
- 출력: 리포트(md/json) — 현 수치 vs 타깃 5~10h, 항목별 민감도.
- 보정 패스: 분포가 타깃 벗어나면 EXP 지수(1.85~2.0 범위)·골드/시간·오프라인 효율 등 보정 제안 + 적용(공식 상수, 단위테스트 동기화).

### 2.2 문서/검증
- `docs/planning/05-balance-philosophy.md` "1차" 표기 항목을 시뮬 결과로 갱신(또는 검증됨 표기). 시뮬 사용법 문서.
- 시뮬 단위테스트(결정적 시드 → 분포 재현).
