# 통합 마스터리 (Unified Mastery) — 설계 문서

> 작성일: 2026-05-29 · 대상 슬라이스: PR #72 (예정) · 브랜치: `plan/72-unified-mastery`
> 분류: 교차 메타 시스템 (기존 시스템 심화 + 엔드게임 성장 우물)

---

## 0. 한 줄 요약

각 주요 시스템마다 **무한 "숙련 트랙"**을 두고, 그 시스템을 플레이하면 숙련
경험치가 **자동 적립**되어 트랙 레벨이 무한히 오른다. 트랙 레벨은 (1) 해당
시스템을 심화하는 **로컬 보너스**와 (2) 계정 단위 **월드 파워**로 합류하는
**숙련 점수**를 동시에 준다. 숙련은 **환생·초월로 초기화되지 않는 영구
성장 소스**다.

---

## 1. 목적 / 배경

- **출발점**: "기존 시스템 심화" → "교차 메타 시스템 1개" → 핵심 목적
  **엔드게임 성장 우물** (던전·장비·룬 등을 마스터한 고수가 계속 쌓을 수
  있는 상위 축, "더 할 게 없다" 방지).
- **제약**: 무한 성장([[project-infinite-growth]]), 콘텐츠
  풍부함([[project-content-richness]]), 한글
  문서([[feedback-korean-docs]]), 5-team 충실
  슬라이스([[feedback-substantial-slices]]).

### 1.1 기존 메타와의 차별 (중복 회피)

이미 다음이 구현되어 있어, 마스터리는 반드시 이들과 직교해야 한다.

| 기존 시스템 | 성격 | 마스터리와의 관계 |
| --- | --- | --- |
| 환생 (Rebirth, PR #19/#46) | 1차 prestige, 리셋 + 영구 포인트 | 마스터리는 점진·영구·활동 기반 (직교) |
| 초월 (Transcendence, PR #47) | 2차 prestige, 깊은 리셋 + 글로벌 배수 `1+count×0.25` | 마스터리는 리셋을 *견딘다* (우물의 핵심) |
| 전투력 (Combat Power, PR #49) | 파생 스탯 집계 **읽기 전용 표시 지표** | 마스터리는 실제 성장 소스 → CP를 *올린다* |
| 무한의 탑 (PR #50/#51) | 콘텐츠 트랙 | 심연 숙련의 XP 소스로 재활용 + 영구 진행 추가 |

> 핵심: 환생·초월은 *리셋형* prestige, CP는 *표시 지표*다. 마스터리는
> "각 시스템을 계속 써서" 오르는 **비(非)리셋 성장 소스**이며, 리셋을 견뎌
> 살아남기에 진짜 "성장 우물"이 되고 리셋 손실감을 완화한다.

## 3. 숙련 트랙 (초기 6트랙)

각 트랙은 무한 레벨. 전역 기여를 서로 다른 능력치에 매핑해 시스템마다
월드 파워 보상의 색깔이 다르게 느껴지도록 한다 (초월의 단조 배수와 차별화).

| 트랙 | XP 소스 (활동) | 로컬 보너스 (해당 시스템 심화) | 전역 매핑 |
| --- | --- | --- | --- |
| **전투 숙련** | 몬스터 처치·스테이지 진행 | 스테이지 보상↑ | 공격력 % |
| **장비 숙련** | 강화·잠재 재감정·세트 완성 | 강화 성공률 보정·자원 절약 | HP/방어 % |
| **심연 숙련** | 던전 클리어·탑 등반 | 던전 보상·입장 횟수↑ | 드롭률 % |
| **룬 숙련** | 룬 합성·장착·세트 | 룬 효과 강화 | 크리티컬 % |
| **야성 숙련** | 펫 먹이·진화 | 펫 효과↑ | 골드/EXP % |
| **탐험 숙련** | 퀘스트·업적 달성 | 보상 획득량↑ | 전 능력치 % |

> 트랙별 로컬 보너스는 V1에서 **트랙당 1종**으로 제한한다 (스코프 통제,
> §7). 추가 로컬 보너스는 후속 PR로 확장.

## 4. 월드 파워 집계 & 능력치 적용

- **월드 파워(World Power)** = 전 트랙의 숙련 점수 합산. HUD 상단(전투력 CP
  옆)에 표시.
- 전역 보너스는 `AIdleCharacter::RefreshDerivedStats`의 **단일 지점**에서
  적용한다 (초월 배수 적용 패턴과 동일). → CP에 자동 반영되고, 다른 성장
  소스와 깔끔히 합성된다.
- 초월 배수(`1+count×0.25`, 코어 스탯 곱)와 **카테고리를 분리**하거나 별도
  가산항으로 두어 이중 계산을 방지한다.

### 4.1 적용 순서 (RefreshDerivedStats 내)

1. 1차/2차 스탯 + 장비/세트/affix/유니크 trait 합성 (기존)
2. 환생 보너스 (기존)
3. 초월 배수 (기존, 코어 스탯 곱)
4. **월드 파워 전역 보너스 (신규)** — 트랙별 매핑 능력치에 가산/배수
5. Combat 초기화 (기존)

## 5. 무한 곡선 철학

- **XP 요구치**: 기하 증가 `XPReq(L) = base × growth^L` → 무한하되 점점
  느려지는 방치형 표준.
- **전역 보너스**: 로그/제곱근형 체감 `bonus = k × ln(1 + level)` → 캡 없이
  무한, 폭주 없음.
- **로컬 보너스**: 선형이되 안전 상한이 필요한 곳만 클램프 (예: 강화 성공률
  보정 상한).
- balance 문서에 초기/중반/엔드 성장 예시를 기록한다.

## 6. 통합 지점 (구현 개요)

| 파트 | 작업 |
| --- | --- |
| **character (메인)** | GameInstance 마스터리 상태, 기존 시스템 XP 훅(처치·강화·던전클리어 등), `RefreshDerivedStats` 단일 지점 전역 보너스 적용, Automation |
| **backend (보조)** | `server/src/core/formulas/mastery.ts` 미러 + `server/src/modules/mastery` 영속/검증, parity 테스트 |
| **designer (보조)** | 숙련 패널(트랙별 레벨·XP바·로컬 보너스·기여 점수) + 헤더 월드 파워, ko/en 로컬라이즈 |
| **balance (보조)** | XP/보너스 곡선 근거 + 성장 단계별 예시 문서 |
| **qa (보조)** | XP 훅 정합·전역 보너스 단조 증가·리셋 생존·CP 증가 E2E 시나리오 |

### 6.1 세이브

- SaveVersion **12 → 13** 상향. 마이그레이션 시 전 트랙 레벨/XP = 0,
  월드 파워 = 0으로 기본값.
- 마스터리는 **계정 단위 영구 저장** (환생/초월 리셋 대상에서 제외).

- 숙련 전용 재화/상점

- 서버 권위 마스터리 (V1은 클라 표시 + 서버 공식 미러)
