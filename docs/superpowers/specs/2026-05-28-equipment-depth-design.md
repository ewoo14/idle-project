# 설계 — 장비 심화 (강화 리스크 + 잠재 레이어)

> 작성일: 2026-05-28 · 상태: 설계 확정 (구현 계획 대기)
> 범위: 단일 통합 슬라이스 (5팀 충실 PR) · 아키텍처: A안 — 클라 게임플레이 + 서버 공식 미러 + 패리티 테스트 + balance-sim

---

## 1. 목표 / 범위

방치형 12+ 톤을 유지하면서 **이미 보유한 장비와 계속 상호작용할 이유**를 만든다. 두 개의 직교 축을 한 슬라이스로 추가한다.

- **수직 긴장 (강화)**: 고단계 강화에 단계 하락 리스크 + 보호서 + 천장(pity)을 도입해, 골드/자원의 깊은 싱크와 긴장감을 만든다.
- **수평 추구 (잠재)**: 드롭 어픽스와 분리된 **별도 잠재 레이어**를 큐브로 리롤해, "완벽 옵션 추구"라는 끝없는 목표를 만든다.

### 비범위 (Non-goals)

- **영구 아이템 파괴 없음** (방치형 톤 유지).
- **서버 권위 지출 검증 없음** — 형제 시스템(스탯 포인트 PR #34, 스킬 랭크 PR #27)과 동일하게 클라이언트 게임플레이 + 서버 공식 미러로 출발. 전 경제 시스템의 서버 권위화는 별도 후속 슬라이스에서 일괄 처리.
- 드롭 어픽스(PR #40/#58) 풀·확률·수치는 **변경하지 않는다**.
- 강화 비용 곡선·성공률 곡선·`1 + EnhanceLevel*0.1` 보상 배율은 **변경하지 않는다** (balance-sim 기준선 보존).
- 가챠/유료 자원 없음 (MVP 규칙).

## 2. 강화 리스크 모델 (스타포스 스타일)

기존 비용·성공률 곡선은 그대로 두고, 그 위에 **단계 하락 리스크 차원**만 얹는다.

### 2.1 리스크 구간

| 구간 | 단계 | 실패 결과 |
| --- | --- | --- |
| **안전 (Safe)** | +0 ~ +9 | 단계 유지 (현행, 골드/자원만 소모) |
| **위험 (Risk)** | +10 ~ +50 | **단계 -1** (보호서 사용 시 단계 유지) |

- 안전 구간 경계 **+9 / +10**, 위험 구간 하락량 **-1**은 1차 시안. balance-sim 결과로 보정.
- 성공률은 기존 `clamp(0.95 - CurrentLevel * 0.018, 0.05, 0.95)` 유지. 비용은 기존 `100 * (CurrentLevel + 1)^2 * RarityCostMultiplier` 유지.

### 2.2 보호서 (Protection Scroll)

- 위험 구간 강화 **실패 시** 1개 소모하여 단계 하락을 방지(단계 유지).
- 시도 **전** UI 토글로 사용 여부 결정(보유 시에만 활성). 성공 시 소모 안 함, 실패 시에만 소모.
- 안전 구간에서는 사용 불가/무의미(하락이 없음).

### 2.3 천장 (Pity)

- 위험 구간에서 **같은 단계 연속 실패 N회**(시안 **12회**) 시, 다음 시도를 강제 성공 처리한다.
- 연속 실패 카운터 `EnhanceFailStreak`를 아이템에 저장한다.
- **리셋 조건**: 강화 성공 시, 또는 단계가 변경될 때(하락 포함) 0으로 리셋.
- 보호서로 단계 유지된 실패도 연속 실패 카운트에 포함한다(천장은 "운빨 보호" 목적이므로 실패 횟수 자체를 센다). — 1차 시안, balance-sim으로 검증.

### 2.4 실효 비용 영향

단계 하락 도입으로 +50 도달 **실효 기대비용이 크게 상승**한다. balance-sim을 신규 실패 모델로 재실행하여 등급별 +50 도달 기대비용·기대시간을 재산출하고, **첫 환생 5~10h 타깃 밴드가 유지됨**을 첨부한다(강화는 1차 출시 후반 롱테일이라 첫 환생 구간에는 영향이 없을 것으로 예상하나 수치로 확인).

## 3. 잠재 레이어 (별도 · 큐브 리롤)

드롭 어픽스(`AffixBonus`: CritRate/AtkSpeed/MagicAtk/PhysDef/MagicDef/Hp/CritDmg)는 **건드리지 않는다**. 그 위에 독립적인 잠재 레이어를 추가한다.

### 3.1 부여 조건 / 등급 / 줄 수

- **부여 조건**: Rare 이상 아이템만 잠재 슬롯 보유. Common/None은 잠재 없음(어픽스 게이팅과 일관).
- **잠재 등급**: `None → Rare → Epic → Unique → Legendary` (5단계).
- **잠재 등급 상한은 아이템 등급에 연동**(끝없는 성장 원칙 — 고등급 아이템일수록 높은 천장):

| 아이템 등급 | 잠재 등급 상한 |
| --- | --- |
| Common / None | 없음 |
| Rare | Epic |
| Epic | Unique |
| Unique | Legendary |
| Legendary / Transcendent / Mythic | Legendary |

- **줄 수 = 잠재 등급**: Rare 1줄 / Epic 2줄 / Unique 3줄 / Legendary 3줄. (1차 시안)

### 3.2 잠재 옵션 풀 (% 기반)

장르 관례대로 **% 승수 옵션**을 사용한다. 유니크 trait가 이미 %승수를 `DeriveStats`에 태우므로 동일 경로를 재사용한다.

| 잠재 옵션 | 비고 |
| --- | --- |
| PhysAtk % | 물리 공격 % |
| MagicAtk % | 마법 공격 % |
| AllStat % | 1차 능력치 전체 % |
| CritRate (p.p.) | 치명 확률 |
| CritDmg (p.p.) | 치명 피해 |
| HP % | 최대 체력 % |
| Defense % | 물리/마법 방어 % |
| GoldGain % | 골드 획득 % |
| DropRate % | 드롭률 % |

- 수치 폭은 **잠재 등급이 높을수록 상향**(예: Rare < Epic < Unique < Legendary). 구체 롤 범위 테이블은 구현 계획에서 확정하고 balance-sim에 반영.
- 한 아이템의 잠재 줄은 옵션 종류 중복 없이 뽑는다(어픽스 롤이 이미 unique-type 방식).

### 3.3 PowerScore / 자동장착 처리

- 잠재 줄은 **실제 전투력(`DeriveStats`)에 반영**된다(% 승수 경로).
- **자동장착 PowerScore에도 가중 합산**한다(어픽스와 동일한 근사 환산 방식). 구체 환산 가중치는 구현 계획에서 확정하되, 클라 `FItemPowerScore::Compute`와 서버 `computeItemPowerScore`가 **동일 항**을 가진다.
- 추가로 **아이템 잠금(`bLocked`) 플래그**를 둔다. 잠긴 아이템은 자동장착이 교체하지 않는다 → 투자한 빌드 보호.
- (기각된 대안: 잠재를 PowerScore에서 제외 — 세트처럼. 이러면 자동장착이 투자템을 더 높은 베이스 PowerScore의 신규 드롭으로 교체할 위험이 있어 비추천.)

## 4. 경제 / 화폐 (신규 자원)

가챠·유료 없음 유지. 신규 자원은 드롭·던전·보스·골드 상점 교환으로 수급한다(GDD §5 경제 일관).

| 자원 | 용도 | 수급 경로 |
| --- | --- | --- |
| **보호서 (Protection Scroll)** | 위험 구간 강화 하락 방지 (실패 시 소모) | 저확률 드롭 + 던전/보스 보상 + 골드 상점 교환 |
| **재설정 큐브 (Reset Cube)** | 같은 잠재 등급 내 줄 옵션 리롤 (저렴) | 드롭 + 골드 상점 교환 |
| **등급 큐브 (Rank Cube)** | 줄 리롤 + 낮은 확률로 잠재 등급 상승 (고가) | 던전/보스 보상 + 골드 상점 교환 |

- 등급 큐브의 등급 상승 확률은 1차 시안으로 낮게 두고(예: 시도당 한 자릿수 %), balance-sim/플레이 체감으로 보정.
- 큐브 2종이 과하면 1종(리롤+상승 내장)으로 통합 가능 — 구현 전 PM 최종 확정 여지로 남김.
- 수급 수치(드롭률, 골드 교환가)는 기존 골드 상점 곡선(PR #38 `300 * RewardMultiplier`)과 정합되게 설정.

## 5. 데이터 / 세이브 / 패리티 아키텍처

### 5.1 아이템 구조체 확장

`FItemInstance`(클라) + 서버 미러에 다음 필드 추가:

- `EPotentialGrade PotentialGrade` (None~Legendary)
- `PotentialLine1 ~ PotentialLine3`: 각 `{ EPotentialStat Stat, float Value }`
- `int32 EnhanceFailStreak` (천장 카운터)
- `bool bLocked` (자동장착 보호 잠금)

### 5.2 플레이어 자원

세이브의 자원 카운트에 `보호서`, `재설정 큐브`, `등급 큐브` 추가.

### 5.3 세이브 마이그레이션

- SaveVersion **11 → 12**.
- 구 세이브 로드 시 신규 필드 기본값으로 채움: `PotentialGrade=None`, 줄 없음(Stat=None), `EnhanceFailStreak=0`, 신규 자원 `0`, `bLocked=false`.
- `CloudSavePayloadMapper` + 서버 세이브 페이로드에 신규 필드 반영(누락 시 기본값으로 안전 처리).

### 5.4 공식 모듈

- **클라**: `EnhanceFormula.cpp/.h` 확장(구간 판정/하락/천장/보호 분기), 신규 `PotentialFormula.cpp/.h`(잠재 부여·등급 게이팅·줄 롤·큐브 리롤·등급 상승).
- **서버**: `enhance.ts` 확장(downgrade/pity/protection 헬퍼), 신규 `potential.ts`.
- **RNG**: 전부 주입 가능한 시드 스트림(`FRandomStream` / `DropRng`)으로 — 결정적 패리티 테스트 가능(어픽스 롤이 이미 이 방식).
- **PowerScore**: `FItemPowerScore::Compute` + `computeItemPowerScore`에 잠재 가중 합산 항을 양쪽 동일하게 추가.

## 6. UI / UX (전부 한글)

### 6.1 강화 패널 확장

- 구간 표시(안전/위험 배지), 위험 진입 시 **"실패 시 -1단계" 경고**.
- 보호서 토글(보유 N개, 실패 시 소모 미리보기).
- 천장 진행도(연속 실패 X/12), 성공률 표시.

### 6.2 잠재 패널 (신규)

- 현재 잠재 등급 + 줄 표시(등급별 색상).
- `재설정 큐브` / `등급 큐브` 사용 버튼(보유 수 표시).
- **결과 미리보기 → 적용 / 유지** 방식(보류 중 롤을 UI 상태로 들고 선택) — 장르 표준 + 플레이어 agency. (즉시 교체가 낫다면 구현 계획에서 변경 가능.)

### 6.3 인벤토리

- 아이템 잠금(🔒) 토글.

### 6.4 디자이너

- design-system 토큰/타이포 재사용, 잠재 등급별 색상, 한국 ERP/메이플 컨벤션 카피, 한글 로컬라이즈(`UI.csv` 등).

## 7. 테스트 / 밸런스 게이트

### 7.1 패리티/단위 테스트 (클라 AutomationSpec ⟷ 서버 vitest, 동일 앵커)

- 강화 하락(위험 구간 실패 -1), 보호서 소모 시 단계 유지, 천장 12회 강제 성공·리셋 조건.
- 잠재 등급 게이팅(아이템 등급별 상한), 줄 수 by 등급, 결정적 시드 롤 분포, 등급 큐브 상승 확률.
- 잠재 포함 PowerScore 클라/서버 일치, 잠금 아이템 자동장착 제외.
- 세이브 마이그레이션 v11→v12(구 세이브 → 신규 필드 기본값).

### 7.2 balance-sim

- 신규 강화 실패 모델(하락+보호+천장)로 재실행 → 등급별 +50 도달 기대비용·기대시간 재산출.
- **첫 환생 5~10h 밴드 유지** 첨부.

### 7.3 QA 시나리오

- 보호서 유/무 위험 강화, 천장 도달, 잠재 등급업(재설정 vs 등급 큐브), 잠금 아이템이 자동장착에서 생존.

### 7.4 문서 동기화

- `05-balance-philosophy.md`: 강화 리스크/잠재 수치 절 추가.
- `01-game-design.md`: §4.3(강화), §4.4(잠재) 갱신.
- 슬라이스 문서(`docs/planning/slices/NN-equipment-depth.md`) 작성.

## 8. 미확정 (구현 계획에서 확정)

- 잠재 옵션 등급별 롤 범위 테이블(구체 수치).
- 잠재 → PowerScore 환산 가중치(구체 계수).
- 보호서/큐브 드롭률·골드 교환가(구체 수치).
- 등급 큐브 등급 상승 확률(구체 %).
- 큐브 2종 유지 vs 1종 통합 최종 결정.
- 잠재 패널 "미리보기 후 적용" vs "즉시 교체" 최종 결정.
